2b. **Lock the control plane read-only (v0.8 provision-or-refuse: PROVISION the local lockdown).** The preflight also REFUSES to start when the gate's own config / hooks / leases are writable by the agent (a bypassed gate could otherwise disable itself; this is the `controlPlaneWritable` refusal). Provision the lockdown now: run `node ${CLAUDE_PLUGIN_ROOT}/hooks/harden-control-plane.mjs` from the repo root. Materialize-then-lock is mandatory and idempotent: you cannot read-only-protect a file that does not exist (CVE-2026-25725: a sandbox could not RO-bind a missing `settings.json`, so the agent created it and injected a hook). It probes exactly the set the preflight's `probeControlPlaneWritable` probes, so after it runs that probe reads `controlPlaneWritable:false`. The hook is fail-closed (non-zero exit if any target could not be locked) but this init step does not abort on a partial lock: print the per-target result and the chown guidance and continue. Durable ownership by a non-agent principal is the real barrier; the chmod/chattr the agent set, a root-equivalent agent could in principle reset, which is why the guidance is to chown and re-run as that owner.
